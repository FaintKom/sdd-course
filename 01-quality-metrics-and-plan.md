# SDD-курс «Управляемая разработка с ИИ» — план разработки и метрики качества

## Задача
Собрать самодостаточный HTML-разбор курса как LXD: как студенты работают, что готовить
к каждому этапу, цели, образовательные результаты (ABCD), артефакты, когнитивная нагрузка,
оценка. Источник правды — гугл-таблица курса (24 вкладки) + опрос студентов (анкета).
Действую как LXD, решения принимаю сам на фактах. Во внешние источники не пишу.

## LXD-линзы (подняты как скиллы)
- **ADDIE** (Branch 2009; Dick&Carey 2014) — 5 фаз, сквозной alignment objective→assessment→content.
- **Cognitive Load Theory** (Sweller 1988/2019; Kalyuga 2003) — intrinsic/extraneous/germane,
  worked-example fading, expertise reversal.
- **Action Mapping** (Moore 2017) — поведение-first, goal→actions→filter (skill vs env/motiv),
  практика > контент.

## Источник правды (только из материалов)
- **Канон программы** (вкладка «ТЗ на ленд», Экран 8): 8 недель, живые воркшопы, старт июль.
  W1 Почему промпты ломаются · W2 От задачи к спецификации · W3 Управление контекстом ·
  W4 Постановка задач AI-агенту · W5 Контроль качества AI-кода · W6 Тестирование AI-разработки ·
  W7 Легаси и существующий код · W8 Сквозной проект и защита.
- **Формат** (вкладка «1. Концепция»): воркшопы = повторение за экспертом + сквозной проект;
  на каждый этап практика; финал — репозиторий с полным циклом + защита перед экспертом.
- **Полный цикл** (Экран 5): задача→спецификация→контекст→реализация→тесты→ревью→доработка→защита.
- **Слепок рынка** (поведенческий источник для action-mapping): задачи junior (5) и middle (5)
  с подзадачами, частотой, приоритетом, инструментами, ожидаемым результатом.
- **ЦА**: junior+ → senior/lead; +QA automation, tech lead, тех-фаундер. НЕ для: не-кодеров,
  не читающих код, без Git, ищущих «приложение без кода».
- **Пререквизиты**: код на 1+ языке, структура проекта, Git (коммиты/ветки/PR), постановка задач,
  чтение чужого кода, база тестов, code review, готовность к Cursor/Claude Code/Copilot/ChatGPT.
- **Инструменты из вкладок**: Cursor, Claude Code, GitHub Copilot, ChatGPT/Codex, AGENTS.md,
  CLAUDE.md, .github/copilot-instructions.md, Spec Kit, Gherkin/Given-When-Then, ADR, Mermaid,
  PyTest/Jest/JUnit/Vitest/Playwright, ESLint/Ruff, GitHub Actions, coverage.
- **Конкуренты**: Udemy SDD (Skliar, 3.5ч, $11–25, Specify-Plan-Tasks-Implement); deeplearning.ai
  SDD with coding agents; Drim Dev (6 нед, 35 000 ₽); Спектр AI-driven (9 нед, 25–60 000 ₽);
  Нейроцех (воркшоп 2 дня). Отстройка: системный spec-driven, не вайбкодинг, воркшопы + сквозной
  проект, контроль качества + легаси.
- **Реальные метрики групп** (вкладка 7): NPS, COR (70% занятий, 50% ДЗ), COR вебинары, COR ДЗ,
  точка падения посещаемости, оценка ДЗ, доходимость до проекта, % достигших целей.

## Ключевые инсайты из материалов (влияют на дизайн)
- **Отзывы прошлых курсов (вкладка 8) — главный риск нагрузки**: «половину занятия скачивали
  образы и боролись с настройкой среды»; «заранее не объявили, что установить»; «настроить среду
  заранее». → EXTRANEOUS load. Митигация: pre-work окружения + «environment doctor» + снапшот-ветки.
- Просили: практику > теории; теория ДО практики; градиент простого→сложного; реальные примеры,
  не абстракции; локальное, не облако (РФ-контекст) → перекликается с опросной болью «дорогие токены».
- **Опрос студентов**: боли — агент сносит код / не знает контекст / галлюцинирует / нет системы;
  безопасность (слив данных, MCP, консоль); дорогие токены / локальные модели; курсы поверхностны.

## Решения по конфликтам источников
- **8 недель (ленд) vs OTUS-шаблон 3–4 мес**: канон = 8 недель воркшопов (продуктовая концепция
  курса перекрывает дефолтный шаблон таблицы). Шаблон используется как операционная рамка
  (структура ДЗ, критерии, преподавательская).
- **Доп.модули из прошлой сессии (дизайн-система, безопасность, локальные модели)**: НЕ в каноне
  ленда. Внесены как опциональные enrichment-блоки, помечены «из опроса, вне канона».

## Метрики качества (PASS/FAIL, итерирую пока не ≥9/10)
1. **Точность источников** — каждый факт прослеживается к таблице/опросу; конфликты разрешены явно; ничего не выдумано.
2. **LXD-полнота** — все 5 фаз ADDIE; измеримые ABCD-результаты на каждую неделю; assessment к каждому результату; обоснование sequencing.
3. **Когнитивная нагрузка** — каждый блок разобран intrinsic/extraneous/germane + тактики; риск «setup съел воркшоп» закрыт явно.
4. **Ясность работы студента** — на каждом этапе: что делает студент, pre-work, подготовка фасилитатора, цель, результат, артефакт; почасовой run-of-show.
5. **Дифференциация ЦА** — гейт по участию (не по «вкусу»); тиры T1–T3; expertise reversal учтён; junior+→senior обслужены.
6. **Action-mapping** — поведение-first; goal→actions→filter (skill vs env/motiv); практика>контент; env-гэпы (setup, платные токены) вынесены в не-тренинговые фиксы.
7. **Артефакты** — конкретные деливераблы на каждой неделе, замаплены на портфолио/резюме и сквозной проект.
8. **Самодостаточный HTML** — один файл, рендерится офлайн, sticky-TOC, печатается, профессионально.
9. **Оценка** — Kirkpatrick L1–L4 + leading indicators, замаплено на реальные метрики групп (COR, NPS, доходимость, % целей).
10. **Actionability** — по доку команда может провести курс: run-of-show, чек-листы фасилитатора, риски+митигации, материалы-чеклист, прототип.

Цель: ≥9/10 PASS, ни одной критической FAIL.
