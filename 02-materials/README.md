# Стартовый пакет материалов курса «Управляемая разработка с ИИ»

Готовые usable-материалы из чек-листа §9 деливербла `../sdd-course-design.html`.
Собраны прототип-first: сначала ядро ценности (W2 спека + W5 ревью) и главная митигация
риска (pre-work окружения), затем контекст (W3) и защита (W8).

## Файлы

| Файл | Неделя | Кому | Назначение |
|---|---|---|---|
| `pre-work-environment-checklist.md` | до W1 | студент | поднять окружение ДО курса; закрывает риск «setup съел занятие» |
| `spec-template.md` | W2 | студент | шаблон спецификации (5 секций) + упражнение «с/без спеки» |
| `context-AGENTS.sample.md` | W3 | студент | образец контекст-файла (границы для агента) |
| `review-checklist-ai-code.md` | W5 | студент + фасилитатор | чек-лист ревью + набор из 10 подсаженных дефектов |
| `midcourse-checkpoint.md` | W5–6 | студент + ментор | формативная проверка: триаж-светофор, peer-review, корректировка траектории + экономия ревью |
| `defense-rubric.md` | W8 | эксперт + студент | рубрика защиты (6 критериев, T1–T3) + шаблон ОС + Гиббс-рефлексия |
| `bloom-taxonomy.md` | вся программа | методист | таксономия Блума: уровни×типы знания, банк глаголов, разметка ОР по неделям |
| `consultation-policy.md` | вся программа | студент + команда + организация | регламент консультаций по запросу: скоуп, лимиты (2×30 мин), шаблон запроса, роли, SLA |
| `expert-brief.md` | вся программа | эксперт (SME) | выжимка для онбординга эксперта: что за курс, роли эксперт/методист, что готовить (P0–P2), FAQ |

## Связь с дизайном
Каждый файл трасируется к источнику (комментарий в конце файла) и к разделу деливербла.
Полный учебный дизайн, нагрузка, оценка и run-of-show — в `../sdd-course-design.html`.

## Порядок пилота (из деливербла §9)
Протестировать `spec-template.md` + `review-checklist-ai-code.md` на 3 представителях ЦА
(junior+ / middle / senior) до сборки остального. Проверить: понятность шаблонов,
работает ли тренировка на подсаженных дефектах, не съедает ли setup время.
