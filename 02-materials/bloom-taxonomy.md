# Таксономия Блума для курса «Управляемая разработка с ИИ»

> Подробная разметка образовательных результатов по таксономии Блума (ред. Андерсон–Кратволь).
> Цель: проверить, что курс ведёт студента вверх по уровням и не застревает на «понять».
> Таксономия двумерная: **уровень мышления × тип знания**.

## Двумерная сетка

**Уровни мышления (снизу вверх):**
1. **Запомнить** — воспроизвести факты, термины.
2. **Понять** — объяснить смысл, привести пример, классифицировать.
3. **Применить** — использовать процедуру в новой ситуации.
4. **Проанализировать** — разложить на части, найти связи, отличить.
5. **Оценить** — судить по критериям, проверить, раскритиковать.
6. **Создать** — собрать новое целое, спроектировать.

**Типы знания:**
- **Фактический** — термины, факты (что такое спека, EARS).
- **Концептуальный** — модели, принципы (почему промпты ломаются, 4 типа сбоя).
- **Процедурный** — как делать (написать спеку, провести агента, ревью).
- **Метакогнитивный** — управление своим мышлением (где доверять ИИ, что проверять, рефлексия).

## Банк глаголов (для формулировки ОР)

| Уровень | Глаголы (рус) |
|---|---|
| Запомнить | назвать, перечислить, воспроизвести, узнать |
| Понять | объяснить, классифицировать, привести пример, сравнить, интерпретировать |
| Применить | применить, использовать, выполнить, реализовать, составить по шаблону |
| Проанализировать | разложить, отличить, сопоставить, выявить причину, структурировать |
| Оценить | проверить, оценить по критериям, обосновать, раскритиковать, принять/отклонить |
| Создать | спроектировать, собрать, разработать, сформулировать новое, синтезировать |

## Разметка результатов по неделям

| Нед. | Результат (B) | Уровень Блума | Тип знания |
|---|---|---|---|
| W1 | классифицирует сбой агента, отличает доверие от контроля | Понять → Проанализировать | концептуальный + метакогнитивный |
| W2 | составляет спецификацию (EARS, AC, DoD) | Применить → Создать | процедурный |
| W3 | формирует контекст-файл, удерживает границы агента | Применить | процедурный |
| W4 | ставит задачу в scope, ведёт агента, проверяет diff | Применить → Проанализировать | процедурный + метакогнитивный |
| W5 | ревьюит AI-код по чек-листу, ловит дефекты | **Оценить** | концептуальный + процедурный + метакогнитивный |
| W6 | проектирует тест-стратегию, coverage как критерий | Применить → Создать | процедурный |
| W7 | вносит изменение через спеку в легаси без сноса | Создать ← Оценить | процедурный + концептуальный |
| W8 | проводит полный цикл, демонстрирует управление ИИ | **Создать** + Оценить | метакогнитивный (интеграция) |

## Прогрессия (проверка восхождения)

```
W1   Понять / Анализ        ▁▂
W2   Применить / Создать     ▃▅
W3   Применить               ▃
W4   Применить / Анализ      ▃▄
W5   Оценить                 ▆
W6   Применить / Создать     ▃▅
W7   Создать / Оценить       ▆▇
W8   Создать (интеграция)    █
```

Вывод: курс не застревает на «понять» — уже с W2 уходит в Применить/Создать, к W5 даёт **Оценить**
(ядро — ревью качества), финал W8 — **Создать** на интеграции. Прогрессия здоровая.

## Метакогниция (я-слой)
Метакогнитивный тип знания закрывается сквозной рефлексией: W1 «где доверять / контролировать»,
чекпоинт и защита — рефлексия по Гиббсу (что сделал → что чувствовал → что понял → что изменю).
Это поднимает курс с процедурного навыка до управления собственным процессом — ценно для senior-трека.

## Как использовать при разработке уроков
1. Для каждого нового ДЗ/активности сверь: какой уровень Блума тренируем?
2. Если активность застряла на «понять/применить» — добавь шаг «оцени/спроектируй».
3. Глагол результата бери из банка выше — он задаёт уровень и метод проверки.
