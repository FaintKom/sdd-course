# Рубрика защиты финального проекта (W8)

> Артефакт недели **W8**. Студент защищает репозиторий со сквозным проектом перед экспертом.
> Ключевой принцип (вкладка «1. Концепция»): студент показывает **не «вот код», а как он управлял
> разработкой с помощью ИИ**. Оценка дуальная: «принято» / «возврат на доработку» (как шаблон ДЗ, 4.1),
> с уровнями T1/T2/T3 поверх.

## Формат защиты
- 5–7 минут рассказ + 3–5 минут вопросы.
- Репозиторий открыт; виден полный цикл: задача → спека → контекст → реализация → тесты → ревью → доработка.
- Студент ведёт по этапам, показывает артефакты каждого шага.

## Рубрика (6 критериев)

| Критерий | Возврат на доработку | Принято (T1/T2) | Отлично (T3) |
|---|---|---|---|
| **1. Спецификация** | нет или неполная | 5 секций, однозначно ограничивает агента | + годна как командный стандарт (Gherkin/ADR) |
| **2. Контекст** | агент трогал лишнее | границы соблюдены (AGENTS.md/CLAUDE.md) | + переносимые правила репо/монорепо |
| **3. Управление агентом** | видна только генерация | виден план + контрольные точки, отклонены лишние правки | + обоснование решений на развилках |
| **4. Ревью качества** | дефекты не пойманы | чек-лист применён, дефекты устранены | + secure coding, доработан чек-лист под команду |
| **5. Тесты** | нет / красный CI | критичная логика покрыта, CI зелёный | + негативы, regression-сеть |
| **6. Демонстрация управления** | «вот код» | показал, как управлял ИИ по этапам | + рефлексия: ускорение vs тех-долг |

## Решение
- **Принято:** критерии 1–6 на уровне «Принято» или выше.
- **Возврат на доработку:** хотя бы один критерий в «Возврат» — точечный список, что доделать.
- **Уровень:** T1 — процесс пройден, агент под контролем; T2 — + доказан эффект; T3 — + масштабируемо/командный стандарт.

## Шаблон обратной связи (поддерживающий, вкладка 4.1)
```
Сильная сторона: <что получилось — конкретно>.
Зона роста: <что улучшить — критерий + как>.
Следующий шаг: <одно конкретное действие>.
Решение: принято / возврат на доработку (уровень T_).
```

## Что проверяющий смотрит в репозитории
- [ ] Спека в репо (не «в голове»), ссылка из PR
- [ ] AGENTS.md/CLAUDE.md с границами
- [ ] История PR: видно, как diff корректировался под спеку
- [ ] Заполненный чек-лист ревью
- [ ] Тесты + зелёный CI
- [ ] README: что делает, как запустить, как проверялось
