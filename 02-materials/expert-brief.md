## 1. Курс

**О чём.** 8 недель, blended-формат: учим разработчиков управлять AI-агентами через
спецификацию и контроль качества (SDD-фреймворк). Большая идея курса:
*«Ты управляешь ИИ через спецификацию и контроль качества, без бессистемного использования промптов»*.

**Как устроен.** Модуль = неделя: асинхронная теория до воркшопа (40–75 мин:
лонгриды + квизы + мини-практики) → **live-воркшоп 2 ч** (ядро недели) → ДЗ = шаг сквозного
проекта (1–4 ч) → рефлексия. Студент весь курс ведёт **свой проект** (фича / багфикс /
рефакторинг) и на W8 защищает репозиторий с полным циклом перед экспертом.

## 2. Программа по неделям (что преподаём и какие инструменты)

| Нед. | Тема | Инструменты, которые изучаем | Артефакт студента |
|---|---|---|---|
| W1 | Почему промпты ломаются: 4 типа сбоя агента; модель «менеджер отделов» | — (концепты) | карта доверия/контроля + тема проекта |
| W2 | От задачи к спецификации (требования, AC, DoD, тесты) | Markdown-шаблон спеки, Gherkin/Given-When-Then, Spec Kit | спека (5 секций) |
| W3 | Управление контекстом проекта | AGENTS.md, CLAUDE.md, .github/copilot-instructions.md | контекст-файл репозитория |
| W4 | Постановка задач AI-агенту: план, scope, контрольные точки | Cursor, Claude Code, Copilot agent mode, git diff/PR | agent plan + PR по спеке |
| W5 | Контроль качества AI-кода | review checklist, Copilot/Codex review, ESLint/Ruff | заполненный чек-лист + исправленный PR |
| W6 | Тестирование AI-разработки | PyTest/Jest/JUnit/Vitest/Playwright, GitHub Actions, coverage | тесты + зелёный CI |
| W7 | Легаси и чужой код | git branches/PR, dependency graph, ADR, Mermaid, characterization tests | PR в легаси-репо |
| W8 | Сквозной проект и защита | репозиторий, README, рубрика защиты | репо + защита |

Курс подразумевается **стек-агностичным**: студент работает в своём языке/стеке, инструменты
агентов — любые из ряда Cursor / Claude Code / Copilot (это обсуждаемо).

## 3. Разделение ролей: эксперт vs методист

**Принцип:** методология, структура и форматы уже спроектированы. От эксперта нужны
**содержание и опыт** — примеры из практики, реальные кейсы, код, цифры, истории провалов.

| | Эксперт (SME) | Методист / команда курса |
|---|---|---|
| Структура курса, ОР, рубрики | нужно согласовать | спроектировано и зафиксировано |
| Тезисы каждого урока | раскрывает тезисы в текст лонгрида | тезисы уже написаны в скелетах |
| Примеры, кейсы, скрины | приносит из реальной практики | подсказывает формат и критерии |
| Учебные репозитории, diff'ы, дриллы | собирает (см. п. 4) | ставит ТЗ, проверяет на соответствие ОР |
| Квизы и мини-практики | верифицирует драфты, поправляет спорное | драфты вопросов готовы в скелетах |
| Заведение в LMS, формы, вёрстка | — | методист/платформа |
| Live-воркшопы (8 × 2 ч) | ведёт (план каждого воркшопа готов) | поддержка, тайминг, материалы |
| Проверка ДЗ | возвраты по критериям «принято/доработка» (критерии готовы) | организация потока сдач |
| Консультации | вторая линия: сложные кейсы, office hours (`consultation-policy.md`) | первая линия — фасилитатор |
| Защиты W8 | принимает по готовой рубрике (`defense-rubric.md`) | организация, расписание |

В каждом скелете урока места, где нужен именно эксперт, помечены тегом **[SME-slot]** —
их можно пройти списком или написать контент урока полностью.

## 4. Что конкретно готовит эксперт (по приоритету)

**P0 — долгие артефакты, начинаем сразу:**
1. **Легаси-репо с «минами»** (для W7) — реалистичный код, тесты, хрупкие зависимости,
   задачи трёх уровней сложности. Самый тяжёлый артефакт, от него зависит вся неделя 7.
2. **Учебный репо-шаблон** + 6–8 тем задач для новичков (W1) — на нём строятся демо и
   упражнения недель 2–5 и 8.
3. **Дрилл «agent drift»: 6 кейсов** (W4) — реальные сессии, где агент ушёл от спеки.

**P1 — кейсы и дриллы (по одному комплекту на неделю):**
5 скринов сбоев агента + квиз (W1) · плохая спека с 6 дырами (W2) · «дырявый» AGENTS.md (W3) ·
5 мини-diff с дефектами для дрилла ревью (W5) · 3 лога красного CI (W6) ·
мини-практика blast radius (W7) · 3–5 мини-задач «цикл за 60 минут» (W8).

**P2 — лонгриды:** 1–2 текста на модуль по 8–15 мин чтения. Тезисы, порядок подачи и
примеры-заготовки уже в скелетах (`../08-skeleton/01-module-w1.md` … `08-module-w8.md`) —
задача эксперта «оживить» текст практикой, а не писать с нуля.

**Плюс шаблоны-инструменты:** agent plan (W4), spec-lite на 1 страницу (W8),
CI-workflow (W6) — короткие рабочие документы.

## 5. Живая нагрузка эксперта во время потока

- **8 воркшопов × 2 ч** (план каждого нужно согласовать: тайминг, материалы, worked examples).
- **Защиты W8** — по рубрике из 6 критериев, формат и шаблон обратной связи готовы.
- **Консультации, вторая линия:** асинхронные ответы на эскалации (до 2 рабочих дней),
  опционально групповые часы приёма 60 мин. Лимиты и рамки зафиксированы в
  `consultation-policy.md` — эксперт не превращается в поддержку.
- **Чекпоинт середины курса (W5–6)** — студенты проверяют себя сами + peer-review;
  эксперт/ментор подключается только к проблемным проектам (светофор 🟡/🔴).
- Проверка ДЗ — дуальная («принято / возврат на доработку»), критерии каждой недели нужно
  согласовать; объём зависит от размера потока.

## 6. Частые вопросы эксперта

**— Мне надо придумывать программу?**
Нет. Программа, образовательные результаты, структура уроков, квизы-драфты, рубрики — готовы,
их нужно посмотреть и согласовать. Нужны ваш опыт и материалы из практики по списку п. 4.

**— Надо записывать видео-лекции?**
Нет. Теория — текст. Где нужна динамика (демо прогона, разбор логов) — скринкаст экрана
без «говорящей головы».

**— Сколько писать текста?**
1–2 лонгрида на модуль, 8–15 мин чтения каждый. Тезисы и порядок подачи прописаны сейчас
в документах — можно ориентироваться на них или писать самостоятельно: вы раскрываете их
и добавляете примеры.

**— Какой стек использовать в материалах?**
Любой ваш основной. Курс независимый от стека; примеры в одном стеке + пометки «в вашем стеке
это …» достаточно. Инструменты агентов — те, с которыми реально работаете (Cursor /
Claude Code / Copilot). При необходимости мы можем выбрать фиксированный набор инструментов.

**— Кейсы обязаны быть реальными?**
Желательно реальные (можно обезличенные). Синтетика допустима, если скрины выглядят живыми.

**— Что если студент спросит то, чего нет в программе?**
Действует регламент консультаций: в рамках — материалы курса и сквозной проект; вне скоупа
(карьера, «напиши за меня», глубокий дебаг чужого прода) — вежливый отказ.
Текст регламента готов, ссылаться можно на него.
