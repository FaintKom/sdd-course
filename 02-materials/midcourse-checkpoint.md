# Промежуточная проверка проекта (checkpoint, конец W5 / начало W6)

> Две цели: **(1) скорректировать траекторию** студента до финала; **(2) сократить стоимость ревью** —
> триаж: ментор тратит время только на проблемных, остальное закрывают self- и peer-review.
> Ставится после W5 (уже есть спека, контекст, ≥1 проведённый агентом PR, ревью-чек-лист) и ДО W7,
> чтобы студент успел исправиться.

## Что проверяем (накоплено за W2–W5)
1. **Тема проекта реалистична по объёму к W8** — не раздута и не пуста.
2. **Спека** (5 секций, EARS) есть и однозначна; нет открытых `NEEDS CLARIFICATION`.
3. **Контекст-файл** (AGENTS.md/CLAUDE.md) реально удерживает агента — не трогает запретное.
4. **≥1 задача проведена агентом в scope**, PR соответствует спеке (лишнее отклонено).
5. **Ревью-чек-лист применён** хотя бы раз (есть заполненный).

## Формат (дёшево-сначала)
1. **Self-check** по чек-листу — студент сам, ~10 мин.
2. **Peer-review в парах** — взаимно по тому же чек-листу, ~20 мин. Обратная связь без ментора.
3. **Светофор** по итогу: 🟢 ок / 🟡 риск / 🔴 off-track.
4. **Ментор разбирает только 🟡/🔴** (~15 мин/чел). 🟢 — авто-ок + короткий письменный комментарий.

### Критерии светофора
- 🟢 **ок** — все 5 пунктов выполнены.
- 🟡 **риск** — 1–2 пункта частично; поправимо за неделю.
- 🔴 **off-track** — спека пустая / тема раздута / агент без границ / нет ни одного проведённого PR.

## Корректировка траектории
- **Студенту 🟡/🔴:** один конкретный план исправления (сузить scope / переписать спеку / добить контекст) — до W7.
- **Команде:** агрегат светофоров по группе. Если у многих 🔴 в одном пункте (напр. контекст) — фасилитатор
  добавляет разбор этого места в W6/W7. Чекпоинт = сигнал, куда подкрутить курс.

## Экономия ревью (деньги)
- **Триаж.** Ментор работает только с 🟡/🔴. При доле 🟢 ~50% → вдвое меньше менторо-часов на этом этапе.
- **Ранняя ловля off-track.** Дешевле исправить раздутый scope на W5–6, чем переписывать на защите.
  Меньше крупных провалов на W8 → короче защита → меньше времени эксперта.
- **Peer-review** переносит часть нагрузки на студентов и сам тренирует навык ревью (germane load).
- **Наследование критериев.** Рубрика защиты W8 использует те же критерии → на защите проверяем дельту,
  а не с нуля. Ещё экономия времени эксперта.

## Рефлексия по Гиббсу (3–5 мин, в пару к светофору)
Замыкает цикл Колба (опыт → рефлексия → вывод). Студент отвечает коротко:
1. **Что делал** — какой шаг проекта прошёл (спека/контекст/агент/ревью)?
2. **Что чувствовал** — где буксовал, где было уверенно?
3. **Оценка/анализ** — где агент сбоил и почему (контекст/доверие/хаос/качество)?
4. **Вывод** — что понял про управление ИИ?
5. **План** — одно действие на следующую неделю.

## Артефакт студента
Заполненный checkpoint-чек-лист + светофор + рефлексия по Гиббсу + (если 🟡/🔴) план исправления.

## Тиры
- **T1:** self + peer по шаблону.
- **T2:** + план исправления своей траектории.
- **T3 (senior/lead):** ревьюит чужой checkpoint в роли «тимлида» — доп. ценность сениору и разгрузка ментора.

## Чек-лист checkpoint (выдать студенту)
```
[ ] Тема укладывается в оставшиеся недели (W6–W8), scope реалистичен
[ ] Спека: 5 секций, EARS, нет открытых NEEDS CLARIFICATION
[ ] Контекст-файл есть; агент не трогал запретные файлы в последнем прогоне
[ ] ≥1 PR проведён агентом, соответствует спеке, лишнее отклонено
[ ] Ревью-чек-лист применён хотя бы раз (приложить)
Светофор: 🟢 / 🟡 / 🔴
План исправления (если 🟡/🔴): ____________________
```
