# Pre-work: окружение (сделать ДО первого воркшопа)

> Главная митигация риска курса «setup съел занятие» (прямой урок из отзывов прошлых курсов,
> вкладка 8). Окружение поднимается ДО W1. На воркшопе — только «environment doctor» 15 мин
> на добивание хвостов. Если что-то не поднялось — переключаешься на снапшот-ветку и не выпадаешь.

## Зачем
8 недель — плотно. Каждая минута воркшопа на скачивание образов и борьбу с настройкой украдена
у практики. Поэтому всё ниже — обязательно до старта. Не успел → напиши куратору заранее.

## Чек-лист (отметь каждое)
- [ ] **Git** установлен, `git --version` работает; настроены `user.name` / `user.email`
- [ ] **GitHub-аккаунт** + доступ по SSH или token (умеешь склонировать приватный репо)
- [ ] **Язык + рантайм** твоего стека установлен (Python 3.x / Node LTS / JDK / Go — что у тебя)
- [ ] **IDE** (VS Code / JetBrains / др.) + терминал внутри работает
- [ ] **AI-инструмент** (основной на потоке) установлен и залогинен — см. «Доступ к модели» ниже
- [ ] **Тест-раннер** стека ставится и гоняет пустой тест (PyTest / Jest / Vitest / JUnit / go test)
- [ ] **Репозиторий-шаблон курса** склонирован, ветка `w1` выбрана
- [ ] **Smoke-тест шаблона** прошёл: по README запустил — поднялось без ошибок

## Доступ к модели (закрывает боль «дорогие токены»)
Один из путей — достаточно любого:
- **Основной инструмент потока:** <Cursor / Claude Code> — по инструкции куратора.
- **Бесплатный путь:** GitHub Copilot (free tier) / ChatGPT / Codex.
- **Локальный путь:** локальная модель (напр. через LM Studio / Ollama) — если не хочешь платить за токены.

> Достаточно одного рабочего варианта. Не распыляйся на все сразу (tool sprawl крадёт время).

## Устранение неполадок по ОС
**Windows**
- [ ] WSL2 поставлен (если стек этого требует)
- [ ] Длинные пути включены, антивирус не блокирует репо-папку
- [ ] PATH видит язык/раннер из терминала IDE

**macOS**
- [ ] Xcode CLT (`xcode-select --install`)
- [ ] Homebrew для зависимостей

**Linux**
- [ ] Пакетный менеджер обновлён; права на рабочую папку есть

## Что принести на W1
- [ ] Поднятое окружение (всё выше ✓)
- [ ] **Кандидат на сквозной проект:** реальная или учебная задача (фича / багфикс / рефакторинг),
      которую будешь вести через весь курс. 2–3 предложения описания.

## Если не поднялось
- Снапшот-ветки `w1…w8` репо-шаблона — рабочее окружение на каждую неделю. Сломалось своё →
  переключаешься на снапшот-ветку недели и продолжаешь практику.
- Напиши куратору ДО воркшопа: разберём в чате, не на занятии.
