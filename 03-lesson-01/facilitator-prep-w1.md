# Подготовка фасилитатора к W1

> Что сделать ДО первого воркшопа. Цель — урок идёт по `lesson-plan-w1.md` без сюрпризов,
> setup не крадёт время, у каждого студента к концу есть карта и тема проекта.

## За 1–2 недели до старта
- [ ] Разослать **pre-work** (`../02-materials/pre-work-environment-checklist.md`) + дедлайн.
- [ ] Определить **основной AI-инструмент потока** (Cursor / Claude Code / Copilot) и путь доступа,
      включая бесплатный/локальный (закрыть «дорогие токены»).
- [ ] Подготовить **репозиторий-шаблон** + снапшот-ветку `w1` (рабочее окружение про запас).
- [ ] Собрать **4 кейса-диалога** (`dialogue-cases-w1.md`): 2 провальных, 2 рабочих.
- [ ] Подготовить **шаблон карты доверия/контроля** (`trust-control-map-template.md`).
- [ ] Учебный список тем проекта для тех, у кого нет своей (фича/багфикс/рефактор).

## За 1 день
- [ ] Прогнать репо-шаблон smoke-тестом на чистой машине/ОС (Win/Mac/Linux — где сможете).
- [ ] Проверить, что снапшот-ветка `w1` запускается по README.
- [ ] Слайды/доска: карта курса, 4 типа сбоя, цикл spec-driven.
- [ ] Собрать список «кто не прислал pre-work» → написать лично.

## Чек-лист «на столе» во время урока
- [ ] `lesson-plan-w1.md` (поминутный план)
- [ ] 4 кейса + ключ ответов (какой тип сбоя в каждом)
- [ ] Шаблон карты доверия/контроля (ссылка студентам)
- [ ] Чек-лист окружения + устранение неполадок по ОС (для environment doctor)
- [ ] Снапшот-ветка `w1` (кому раздать при поломке)

## Тайминг-дисциплина (главное)
- Environment doctor — **строго 15 мин**. Глубокие поломки → в чат после урока, не тормозить группу.
- Worked example на кейсах — не уходить в дебаты об инструментах.
- Оставить 15 мин на выбор темы: без темы студент выпадает из сквозного проекта.

## Обратная связь (шаблон, вкладка 4.1)
```
Сильная сторона: <что получилось>.
Зона роста: <что улучшить>.
Следующий шаг: <одно действие к W2>.
Решение: принято / возврат на доработку.
```
