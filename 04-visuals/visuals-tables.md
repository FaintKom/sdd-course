## 1. Карта пути студента (Student Journey Map)

Путь от хаотичных промптов к инженерному управлению AI-разработкой. Кривая уверенности растёт от W1 (хаос, нет доверия к ИИ) к W8 (системный рабочий процесс).

| Нед. | Тема | Что делает | Артефакт | Боль (опрос) | Образ. результат | Эмоция | Мотив / контрмера | 4C/ID |
|---|---|---|---|---|---|---|---|---|
| **W1** | Почему промпты ломаются · менеджер отделов | Классифицирует сбой агента; строит модель «менеджер ↔ отделы»: что отдать ИИ, что проверять | карта доверия/контроля + тема | «всё хаотично, нет системы» | Атрибутирует сбой ≥3/4; распределяет задачи по «отделам» | любопытство + тревога «потяну ли» | рамка + психобезопасность | SI |
| **W2** | От задачи к спецификации | Пишет спеку (EARS, clarification), прогон агента с/без | спека — 5 секций, EARS | «сносит рабочее решение» | Спека покрывает 5 секций; расхождение с/без | фрустрация — **яма #1** (спека трудна) | early-win «с/без спеки» | PI · LT |
| **W3** | Управление контекстом | Собирает контекст-файл (AGENTS.md/CLAUDE.md), границы | AGENTS.md / CLAUDE.md | «не знает контекст проекта» | Агент не выходит за границы, не трогает запретное | уверенность растёт | сквозной проект как якорь | PI |
| **W4** | Постановка задач агенту | Ставит задачу в scope, ведёт агента по плану с контрольными точками | agent plan + PR | «лишние правки, выход за scope» | diff соответствует спеке, лишнее отклонено | ощущение контроля над агентом | цикл план→diff→коррекция | LT |
| **W5** | Контроль качества AI-кода | Ревьюит AI-код по чек-листу на подсаженных дефектах | чек-лист ревью + PR | «галлюцинирует, сложно ревьюить» | Ловит ≥80% типовых дефектов до мерджа | «ага!» + тревога показа ревью | blameless + подсаженные дефекты | PP · PI |
| **W6** | Тестирование AI-разработки | Пишет тесты из EARS, негативы, coverage, CI | тесты + coverage, CI | «доверять нельзя без проверки» | Критичная логика покрыта, CI зелёный | рутина / риск скуки | T3-стретч от скуки | PP |
| **W7** | Легаси и существующий код | Фича/багфикс/рефактор через спеку в чужом коде | PR в легаси-репо | «как подключать ИИ к легаси» | Тесты зелёные, смежное не сломано | фрустрация — **яма #2** (чужой легаси) | репо заранее, поздняя постановка | LT |
| **W8** | Сквозной проект и защита | Собирает полный цикл, защищает перед экспертом | репозиторий + защита | «нет доказательства навыка» | Полный spec-driven цикл на защите | тревога защиты + гордость | защита, не экзамен | LT |

**Пример сквозной задачи** (одна тема через весь курс): «эндпоинт отмены заказа, окно 30 мин» в REST-API заметок/задач или Telegram-боте → спека с EARS → AGENTS.md (запрет менять схему БД, только `services/`) → реализация в scope `order_service.py` → ревью PR с подсаженными дефектами → тесты (happy + просрочка окна + пустой ввод) → багфикс в легаси → защита.

> **Желательное (опц.):** дизайн-система как спека для UI — не отдельная неделя, врезка в W3–W4 для frontend/fullstack (токены, компоненты, правила = контекст для агента). Другие опции v2: безопасность AI-разработки, локальные модели / стоимость.

## 2. Граф программы (модули → финальный проект → образовательные результаты)

8 модулей складываются в сквозной spec-driven цикл и сходятся в финальном проекте, который доказывает 6 образовательных результатов. Цвет/тег узла — доминирующий компонент 4C/ID.

### 2.1 Спина программы (8 модулей)

| Модуль | 4C/ID | Тема |
|---|---|---|
| W1 | SI | Почему промпты ломаются + менеджер отделов |
| W2 | PI / LT | Спецификация (EARS, clarify) |
| W3 | PI | Контекст проекта |
| W4 | LT | Постановка задач агенту |
| W5 | PP / PI | Контроль качества |
| W6 | PP | Тестирование AI-разработки |
| W7 | LT | Легаси и существующий код |
| W8 | LT | Проект + защита |

Поток: W1 → W2 → W3 → W4 → W5 → W6 → W7 → W8 → **Финальный проект (LT)** — полный spec-driven цикл: задача → спека → контекст → реализация → тесты → ревью → доработка → защита.

### 2.2 Финал доказывает 6 образовательных результатов

| ОР | 4C/ID | Что выпускник делает |
|---|---|---|
| **ОР-1** | LT | Полный spec-driven цикл без ручного переписывания |
| **ОР-2** | PI | Формализует задачу в спеку (EARS, clarify) |
| **ОР-3** | PI | Управляет контекстом — агент в границах |
| **ОР-4** | LT | Ставит и ведёт агента в scope, diff по спеке |
| **ОР-5** | PP | Проверяет AI-код тестами + ревью, ловит ≥80% |
| **ОР-6** | LT | Применяет в легаси, встраивает процесс в команду |

**Легенда 4C/ID:** LT — learning task (целостная задача) · SI — supportive info (теория, модели) · PI — procedural info (шаблоны, чек-листы) · PP — part-task practice (рекуррентные навыки).
