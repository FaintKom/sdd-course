# План урока W2 — От задачи к спецификации

> **Статус: драфт к согласованию с экспертом** (тайминг, материалы, worked examples).
> Тип 4C/ID: PI + LT. ≈2 ч, live. Артефакт — спецификация. Intrinsic high → worked-example fading.
> Спроектирован по скиллу «Образовательный опыт»: цикл Колба + рефлексия Гиббса, нагрузка по Свеллеру,
> эмоции по Пекруну, мотивационная поддержка, чек-лист конструктивной согласованности.

## Образовательный результат (ABCD)
- **A** студент · **B** составляет спецификацию (требования, ограничения, AC, DoD, тесты) по шаблону ·
  **C** для задачи своего сквозного проекта · **D** покрыты все 5 секций; результат агента с/без спеки
  измеримо расходится.

## Вектор опыта (аффективная цель)
Студент **переживает контроль**: «не я подстраиваюсь под агента — я задаю рамку, и он слушается». Сдвиг установки от «ИИ непредсказуем» к «непредсказуемость = моя нечёткая постановка».

## Конструктивная согласованность (чек-лист)
ОР (составить спеку) ↔ **стратегия** (рабочий пример + практика на своём проекте, Лорильяр: практика+создание) ↔ **оценка** (критериальная: 5 секций + измеримое расхождение с/без). Пространство для смысла: задача — своя.

## Артефакт
Спецификация (5 секций) + заметка о расхождении прогона с/без спеки.

## Pre-work
Прочитать примеры хорошей/плохой спеки (`../02-materials/spec-template.md`); описать свою задачу 3–5 предложениями. *(Эффект взаимосвязанности — стартуем от уже принесённой задачи студента.)*

## Подготовка фасилитатора
Шаблон спеки; пары «хорошо/плохо»; заготовленная задача для живого показа; снапшот-ветка `w2`; Gherkin/ADR для T3.

## Поминутный план (≈120 мин) — разметка по циклу Колба + нагрузка

| Время | Блок | Колб | Что происходит | Нагрузка (стратегия Свеллера) |
|---|---|---|---|---|
| 0:00–0:15 | Environment doctor | — | добор окружения; сломан → ветка `w2` | extraneous ↓ (убрать избыточное) |
| 0:15–0:35 | Концепт: спека = контроль | **AC** | 5 секций + EARS малыми порциями, устно+на экране | intrinsic high → дроблю; модальность |
| 0:35–1:10 | Worked example: пишу спеку вживую, прогон с/без | **CE (vicarious)** | T1 повторяют синхронно → completion частичного шаблона | **рабочий пример** (ядро урока) |
| 1:10–1:40 | Своя практика: спека на свою задачу, прогон с/без | **AE** | эксперт обходит, правит AC/DoD | germane; рост независимости (T2/T3 без шаблона) |
| 1:40–1:52 | Разбор ошибок + рефлексия Гиббса | **RO** | см. блок рефлексии ниже | germane (строит схему «спека→diff») |
| 1:52–2:00 | Закрытие | — | критерий артефакта + pre-work W3 | — |

*Вход не жёстко с CE: T1 входят через рабочий пример (CE), T2/T3 — сразу через свою практику (AE). Стратегию выбирает студент.*

## Рефлексия (Гиббс, кратко, в блоке 1:40)
Описание (что писал) → чувство (где буксовал — «пустой лист»?) → анализ (что изменилось в diff от спеки?) → вывод (спека = рычаг, не бюрократия) → план (что уточню в спеке к W3).

## Эмоциональная динамика (Пекрун)
Доминанта — **фрустрация достижения (−), яма #1**: спека трудна. Тип: эмоция достижения. Контрмера — **early-win** «с/без спеки» (видимое расхождение = активизирующее удовольствие сразу) + fading (нет пустого листа). Цель — преодолеваемая сложность, не пик.

## Мотивационная поддержка
**Ценность:** спека = строка резюме (Spec-Driven Development). **Успех:** микро-победа «агент послушался спеку» в первые 40 мин. **Прозрачность:** студент видит, что это шаг 2 из 8 к своему репо.

## Проверка (формирующая, критериальная)
5 секций покрыты, AC однозначны; зафиксировано **измеримое** расхождение с/без (лишние файлы, додуманные требования).

## ДЗ к W3
Дописать спеку; составить карту репозитория (модули, точки входа, **запретные зоны**) — вход в W3. «Принято»: 5 секций + расхождение. ~2–3 ч.

## Тиры (обратная сторона экспертности)
- **T1:** шаблон + worked example; happy-path фича.
- **T2:** пустой шаблон + самопроверка; своя задача.
- **T3:** без шаблона, Gherkin/ADR; спека на легаси как командный стандарт.

## Риски урока
- Яма #1 (буксует на спеке) → fading + early-win. · Формализм «спека ради спеки» → каждый пункт меняет diff, иначе убрать. · Разброс форматов → один шаблон на поток.
