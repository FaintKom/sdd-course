# План урока W5 — Контроль качества AI-кода

> **Статус: драфт к согласованию с экспертом** (тайминг, материалы, worked examples).
> Тип 4C/ID: PP + PI. ≈2 ч, live. Артефакт — заполненный чек-лист ревью + исправленный PR. Ядро ценности курса.
> Спроектирован по скиллу «Образовательный опыт»: цикл Колба + Гиббс, нагрузка по Свеллеру,
> эмоции по Пекруну (соц. безопасность!), мотивационная поддержка, чек-лист конструктивной согласованности.

## Образовательный результат (ABCD)
- **A** студент · **B** прогоняет AI-сгенерённый код по чек-листу ревью · **C** имея diff и спеку
  (в т.ч. с подсаженными дефектами) · **D** находит ≥80% типовых дефектов до мерджа.

## Вектор опыта (аффективная цель)
Студент **переживает зоркость**: «я ловлю то, что агент додумал». Сдвиг от «AI-код проверять страшно/непонятно как» к «у меня есть оптика — чек-лист и подсаженные дефекты натренировали глаз».

## Конструктивная согласованность (чек-лист)
ОР (ревью по чек-листу) ↔ **стратегия** (частичная практика на подсаженных дефектах — PP; рабочий пример разбора) ↔ **оценка** (критериальная, измеримая: доля найденных дефектов ≥80%). Безопасный смысл — дефекты чужие, не свои.

## Артефакт
Заполненный чек-лист ревью + исправленный PR.

## Pre-work
Прочитать чек-лист (`../02-materials/review-checklist-ai-code.md`); PR из W4 готов к разбору. *(Эффект взаимосвязанности — ревьюим то, что сам провёл.)*

## Подготовка фасилитатора
Чек-лист ревью + **набор из 10 подсаженных дефектов** (галлюцинация API, лишний файл, незаметный регресс); снапшот-ветка `w5`; секундомер на замер доли найденного.

## Поминутный план (≈120 мин) — разметка по циклу Колба + нагрузка

| Время | Блок | Колб | Что происходит | Нагрузка (стратегия Свеллера) |
|---|---|---|---|---|
| 0:00–0:15 | Environment doctor | — | добор окружения; сломан → ветка `w5` | extraneous ↓ |
| 0:15–0:35 | Концепт: где агент врёт | **AC** | оси чек-листа: спека, безопасность, лишнее, тех-долг | intrinsic med; фиксированный чек-лист снижает поиск |
| 0:35–1:05 | Worked example: разбор diff с дефектом | **CE (vicarious)** | эксперт вслух атрибутирует 1 дефект; модель «как смотреть» | **рабочий пример** |
| 1:05–1:35 | Практика: подсаженные дефекты | **AE (PP)** | студент гоняет diff по чек-листу, ловит дефекты; замер доли | germane high (прямой навык ревью) |
| 1:35–1:52 | Свой PR + рефлексия Гиббса | **RO** | переносит навык на свой PR из W4 | germane |
| 1:52–2:00 | Закрытие | — | критерий артефакта + pre-work W6 | — |

*Безопасный порядок: сначала дефекты в чужом коде (PP), только потом — свой PR. Снимает соц. тревогу.*

## Рефлексия (Гиббс, кратко, в блоке 1:35)
Описание (что ревьюил) → чувство (страшно ли показывать свой код?) → анализ (какие дефекты пропустил и почему) → вывод (ревью = успех, не позор) → план (что добавлю в чек-лист к своему стеку).

## Эмоциональная динамика (Пекрун) + психобезопасность
Доминанта — **эпистемическое «ага!»** (пойманный дефект) + **социальная тревога** показа ревью. Контрмеры (CSCI, безопасность #1): blameless-нормы из W1; тренировка на **подсаженных** дефектах (не своём коде); пойманный дефект = успех ревью. Фасилитатор первым показывает свой провальный прогон.

## Мотивационная поддержка
**Ценность:** Code Review AI-кода = строка резюме, ядро найма. **Успех:** измеримое «нашёл 8 из 10» — ощутимая микро-победа. **Прозрачность:** шаг 5/8, это то, ради чего курс.

## Проверка (формирующая, критериальная, измеримая)
Доля найденных подсаженных дефектов **≥80%**; чек-лист заполнен; дефекты в своём PR устранены. *(Ранний прокси освоения ядра — leading indicator.)*

## ДЗ к W6 / Чекпоинт
Ревью своего PR по чек-листу, устранить дефекты. **Промежуточный чекпоинт (W5→W6):** self+peer-review → светофор 🟢/🟡/🔴 (`../02-materials/midcourse-checkpoint.md`). «Принято»: дефекты найдены и устранены. ~2 ч.

## Тиры (обратная сторона экспертности)
- **T1:** ревью по чек-листу на узнаваемых дефектах. · **T2:** свой PR + secure-coding пункты. · **T3:** доработка чек-листа под команду, разбор Copilot/Codex review.

## Риски урока
- Боязнь показать ошибки → blameless + чужие дефекты сначала. · Абстрактное ревью → подсаженные дефекты, не теория. · Чек-лист «для галочки» → замер доли найденного как факт.
