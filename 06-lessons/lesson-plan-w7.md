# План урока W7 — Работа с легаси и существующим кодом

> **Статус: драфт к согласованию с экспертом** (тайминг, материалы, worked examples).
> Тип 4C/ID: LT, high intrinsic. ≈2 ч, live. Артефакт — PR в легаси-репо. **Яма #2 — главный риск отвала.**
> Спроектирован по скиллу «Образовательный опыт»: цикл Колба + Гиббс, нагрузка по Свеллеру,
> эмоции по Пекруну (преодолеваемая фрустрация), мотивационная поддержка, чек-лист согласованности.

## Образовательный результат (ABCD)
- **A** студент · **B** вносит фичу/багфикс/рефакторинг через спеку в существующий код · **C** в легаси-репо,
  не им написанном · **D** изменение проходит тесты, не ломает смежное, ограничено заявленным scope.

## Вектор опыта (аффективная цель)
Студент **переживает перенос**: «я применил весь цикл в реальном незнакомом коде, а не в учебной песочнице». Это момент «я правда умею», ближайший к рабочей реальности.

## Конструктивная согласованность (чек-лист)
ОР (изменение в легаси через спеку) ↔ **стратегия** (целостная учебная задача LT в аутентичном легаси; перенос навыков W2–W6) ↔ **оценка** (критериальная: тесты зелёные, смежное цело, scope соблюдён). Смысл — чужой реальный код.

## Артефакт
PR в легаси-репо: изменение в scope, зелёные тесты, смежное не сломано.

## Pre-work
Клонировать предоставленный легаси-репо; прочитать README/структуру. *(Эффект взаимосвязанности + extraneous ↓: репо выдан заранее, не знакомимся на уроке.)*

## Подготовка фасилитатора
Учебный **легаси-репо с тестами и «минами»** (хрупкие зависимости); снапшот-ветка `w7`; пример ADR; парные группы для разбора (снизить тревогу).

## Поминутный план (≈120 мин) — разметка по циклу Колба + нагрузка

| Время | Блок | Колб | Что происходит | Нагрузка (стратегия Свеллера) |
|---|---|---|---|---|

## Рефлексия (Гиббс, кратко, в блоке 1:40)
Описание (какое изменение внёс) → чувство (паника от чужого кода?) → анализ (где агент сбоил: контекст/доверие/хаос/качество) → вывод (рамки держат даже в незнакомом коде) → план (как подойду к своему рабочему легаси).

## Эмоциональная динамика (Пекрун)
Доминанта — **фрустрация достижения (−), яма #2 — главный риск отвала**: чужой код + ограничения = пик. Контрмеры: репо выдан заранее; **поздняя постановка** (есть беглость); T1 получает **узкий локализованный багфикс**; **парный разбор** (социальная опора). Фрустрация преодолеваемая, без пикового состояния.

## Мотивационная поддержка
**Ценность:** работа с легаси через ИИ = сильнейшая строка резюме, прямой рабочий навык. **Успех:** «не сломал смежное» — ощутимая победа над страхом. **Прозрачность:** шаг 7/8, остался только сбор и защита.

## Проверка (формирующая, критериальная)
PR в легаси: тесты зелёные, смежное не сломано, scope соблюдён.

## ДЗ к W8
Довести репозиторий + README; подготовить 5–7-мин рассказ по этапам цикла (`../02-materials/defense-rubric.md`). «Принято»: тесты зелёные, смежное цело. ~3–4 ч.

## Тиры (обратная сторона экспертности)
- **T1:** локализованный багфикс в выданном репо. · **T2:** фича/рефактор. · **T3:** свой рабочий легаси + бюджет токенов как ограничение.

## Риски урока
- Яма #2 (отвал) → репо заранее + поздняя постановка + парность + узкий T1. · Незнакомый репо (extraneous) → выдан в pre-work. · Агент ломает смежное → разбор «мин» вживую.
