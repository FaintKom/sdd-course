# План урока W8 — Сквозной проект и защита

> **Статус: драфт к согласованию с экспертом** (тайминг, материалы, worked examples).
> Тип 4C/ID: LT, summative. ≈2 ч, live. Артефакт — репозиторий с полным циклом + защита.
> Суммативная демонстрация: не «вот код», а **как управлял ИИ**. Формат отличается от обычного поминутный план — это защита.
> Спроектирован по скиллу «Образовательный опыт»: замыкание цикла Колба, рефлексия Гиббса как часть защиты,
> эмоции по Пекруну (защита = защита, не экзамен), мотивационная поддержка, чек-лист согласованности.

## Образовательный результат (ABCD)
- **A** студент · **B** демонстрирует полный spec-driven цикл и то, как управлял ИИ · **C** на защите перед
  экспертом, репозиторий открыт · **D** проходит рубрику защиты (6 критериев); показывает управление, не только итог.

## Вектор опыта (аффективная цель)
Студент **переживает доказанную субъектность**: «я провёл реальную задачу через весь цикл и могу показать, как именно управлял». Закрытие большой идеи: «управляю через спеку и контроль, а не угадываю промптами».

## Конструктивная согласованность (чек-лист)
ОР (полный цикл + демонстрация управления) ↔ **стратегия** (целостная задача LT + артикуляция-рефлексия) ↔ **оценка** (**суммирующая, критериальная**: рубрика защиты, 6 критериев, T1–T3 — `../02-materials/defense-rubric.md`). Смысл — свой проект, своя история управления.

## Артефакт
Репозиторий: задача→спека→контекст→реализация→тесты→ревью→доработка + защита (рассказ).

## Pre-work
Довести репо и README; подготовить 5–7-мин рассказ по этапам цикла; продумать рефлексию по Гиббсу. *(Сборки финала «с нуля» нет — проект рос всю программу, extraneous ↓.)*

## Подготовка фасилитатора/эксперта
Рубрика защиты (`../02-materials/defense-rubric.md`); расписание слотов; форма поддерживающей ОС (сильные стороны + зона роста + следующий шаг); тон защиты.

## Поминутный план (≈120 мин) — защита + замыкание цикла

| Время | Блок | Колб | Что происходит | Нагрузка / эмоция |
|---|---|---|---|---|
| 0:00–0:10 | Рамка дня | — | «это защита, как управлял ИИ, не экзамен»; снять стресс | соц. тревога ↓ |
| 0:10–1:35 | Защиты по слотам | **AC + RO** | 5–7 мин рассказ + 3–5 мин вопросы; ведёт по этапам цикла, показывает артефакты | germane (артикуляция закрепляет схему) |
| 1:35–1:50 | Финальная рефлексия по Гиббсу | **RO → AE** | что делал → что чувствовал → анализ сбоя → вывод → **план переноса** | метакогнитивный слой |
| 1:50–2:00 | Закрытие курса | — | дуальное решение + следующий шаг каждому; завершающий блок | гордость |

*Цикл Колба замыкается: AE прошлых недель → RO/AC в защите → новый AE = план переноса на свою следующую задачу (обучение продолжается за пределами курса).*

## Рефлексия (Гиббс — часть защиты, 2–3 мин)
1. Что делал — провёл проект через полный цикл. 2. Что чувствовал — где трудно (легаси/ревью/scope). 3. Анализ — где ИИ сбоил, как удержал рамками. 4. Вывод — что понял про управление ИИ (связь с большой идеей). 5. **План переноса** — как применю spec-driven к следующей реальной задаче.

## Эмоциональная динамика (Пекрун)
Доминанта — **тревога достижения (экзамен) + социальная + гордость**. Контрмера — рефрейм **«защита, не экзамен»**; дуальная рубрика (принято/доработка, не оценка-клеймо); поддерживающая ОС; фасилитатор фокусирует на «как управлял», а не «идеален ли код». Финал на гордости (эмоция достижения +).

## Мотивационная поддержка
**Ценность:** сквозной проект на GitHub + защита = главный артефакт портфолио/резюме. **Успех:** видимый полный цикл, собранный своими руками. **Прозрачность:** рубрика известна заранее (наследована с чекпоинта — на защите проверяем дельту).

## Проверка (суммирующая, критериальная)
Рубрика защиты, 6 критериев (спека / контекст / управление / ревью / тесты / демонстрация). Решение: принято / возврат на доработку + уровень T1/T2/T3.

## Уровни (T1/T2/T3)
- **T1:** процесс пройден, агент под контролем. · **T2:** + доказан эффект, пойманы дефекты. · **T3:** + масштабируемо, командный стандарт, обоснование решений на развилках.

## Риски урока
- Стресс «экзамена» → showcase-рамка + дуальная рубрика. · «Вот код» вместо управления → вопросы ведут к процессу. · Перебор по времени → жёсткие слоты, рубрика наследует чекпоинт (проверяем дельту).
