## P0 — Программа и структура

### 1. Раздел 07 «Программа» — заменить все 8 модулей
Сейчас на ленде другая программа (модули про EARS-анатомию, карту инструментов, два модуля легаси, команду). Канон:

| № | Было (ленд) | Должно быть (канон) |

| 01 | Зачем нужны спецификации (и почему сейчас) | Spec-Driven Development: почему промпты ломаются |
| 02 | Анатомия хорошей спецификации | От задачи к спецификации |
| 03 | Контекст и оснастка агента | Управление контекстом проекта |
| 04 | Карта SDD-инструментов | Постановка задач AI-агенту |
| 05 | Проекты с историей. Вход в существующую базу | Контроль качества AI-кода |
| 06 | Проекты с историей. Эволюция через спеки | Тестирование AI-разработки |

К каждому модулю — артефакт недели (из канона §1): карта доверия/контроля → спека (5 секций) → AGENTS.md/CLAUDE.md → agent plan + PR → чек-лист ревью → тесты + зелёный CI → PR в легаси → репозиторий + защита.

### 2. Раздел 06 «Архитектура: 4 слоя» — переписать слои под канон
Сохранить подачу «4 слоя по 2 недели», заменить содержимое:

| Слой | Было | Должно быть |
|---|---|---|
| 1 (W1–2) | Спека без водопадного болота (EARS, Constitution) | **Основы управления:** почему промпты ломаются, ментальная модель «менеджер отделов», задача → спецификация (5 секций, EARS/Given-When-Then) |
| 2 (W3–4) | Оснастка агента (steering, Claude Code, MCP, Spec Kit/Kiro/OpenSpec/BMAD) | **Контекст и ведение агента:** AGENTS.md/CLAUDE.md, границы изменений, план агента, scope, контрольные точки, PR по спеке |
| 3 (W5–6) | Легаси и большие кодовые базы («главный блок») | **Контроль качества:** ревью AI-кода по чек-листу (детект ≥80% дефектов), тест-стратегия, coverage, CI |
| 4 (W7–8) | Команда и quality gates + выпускной | **Легаси и финал:** изменение через спеку в чужом коде, сборка полного цикла, защита перед экспертом |

Убрать заявку «легаси — главный блок, 2 недели»: в каноне легаси = W7 (одна неделя, пик нагрузки).

### 3. Раздел 02 «Резюме» — заменить 5 навыков на ОР канона
Было: «внедряю SDD на базе Spec Kit/Kiro/OpenSpec», «реверс-спецификация и зональные спеки», «CI ловит расхождение спека-кода», «полная оснастка Claude Code».
Проблема: навыки привязаны к инструментам и обещают темы, которых нет в программе (реверс-спека, зональные/дельта-спеки, spec-vs-code CI).

Должно быть (из ОР-1…ОР-6, формулировки от первого лица):
- Провожу задачу через полный spec-driven цикл без ручного переписывания кода за агентом
- Превращаю задачу в спецификацию, которая однозначно ограничивает агента (требования, AC, DoD, тесты)
- Готовлю контекст проекта (AGENTS.md/CLAUDE.md) так, что агент не выходит за границы
- Ставлю задачу агенту в scope и веду по плану — лишние изменения отклоняю
- Ловлю ≥80% типовых дефектов AI-кода (галлюцинации, лишние правки, тех-долг) до мерджа
- Безопасно применяю ИИ в легаси и встраиваю spec-driven в свой workflow — на любом инструменте

### 4. Раздел 09 «Отстройка» — уникальные компоненты
Убрать/заменить: «EARS, дельта-спеки, живые спеки» (дельта-/живых спек в программе нет).
Заменить на канонные дифференциаторы: сквозной проект на своей рабочей задаче, измеримые критерии (≥80% детект дефектов, зелёный CI, рубрика защиты из 6 критериев), работа с легаси (W7), tool-agnostic процесс (Cursor / Claude Code / Copilot).
«Работа с легаси» — оставить. «Закрытый чат бессрочно», «Сертификат» — бизнес-решение, не трогаю.

## P1 — Цифры формата

### 5. Hero: «2×/нед живых занятия» → «8 live-воркшопов по 2 часа»
Канон: 1 воркшоп в неделю × 120 мин. Заодно уходит внутреннее противоречие ленда (hero: 2/нед = 16 занятий; раздел 12: «8 живых сессий»).

### 6. Раздел 12: «8 живых сессий с автором (90 минут)» → «8 live-воркшопов по 120 минут»

### 7. Раздел 12: «25–30 часов видео-уроков» → реальный объём
Канон асинхрона: 40–75 мин/нед (2 коротких урока + квиз/мини-практика до воркшопа) ≈ **6–8 часов** на курс. Подача: «короткая теория до воркшопа — всё остальное руками» (принцип pre-training, 70% практики). 25–30 ч видео противоречит и нагрузке, и обещанию «70% практики».

### 8. Нагрузка «6–8 часов в неделю» → «4–7 часов в неделю»
Канон: 4–6.5 ч/нед, красная линия 7 (пик — W7, легаси). Поправить в разделе 08 («Квалификация») и в FAQ «Сколько часов в неделю?».

## P2 — Формулировки и overclaim

### 9. Раздел 05: «Агент перестаёт галлюцинировать» — снять overclaim
Агент не перестаёт галлюцинировать — студент учится ловить дефекты до мерджа (ОР-5: ≥80% по чек-листу). Замена: «Галлюцинации агента перестают доезжать до мерджа».

### 10. Раздел 05: «Цикл укладывается в один обед» — допустимо, привязать к факту
В каноне есть опора: урок 8.2 «цикл за 60 минут» (мини-задача полным циклом). Оставить, но не обещать это с первой недели.

### 11. Инструментовка: убрать жёсткую привязку к Claude Code / Spec Kit
Ленд продаёт «полную оснастку Claude Code» и «Spec Kit, Kiro, OpenSpec, BMAD» как ядро. Канон (ОР-6): процесс не завязан на конкретный инструмент; рабочий набор — Cursor / Claude Code / Copilot, Spec Kit упоминается как инструмент W2. Инструменты — в раздел «с чем работаем», не в обещания результата.

### 12. Раздел 04 «Узнавание» — сверить триггеры с W1
«Код упал в проде», «диффы на 500 строк» — ок, это модель сбоя агента из W1 (контекст/доверие/хаос/качество). Желательно 4 признака = 4 причины сбоя — усилит связь ленда с программой.

## P3 — Проверить / бизнес-решения (не правки канона)

### 13. POSTMORTEM.md (раздел 03 «Портфолио»)
В материалах курса артефакта POSTMORTEM.md нет (проверено по репо). Либо убрать с ленда, либо добавить в W8 (ДЗ «финализация репо») и в рубрику защиты. Рекомендация: добавить в курс — дешёвый и продающий артефакт, согласуется с рефлексией 8.5.

На ленде нет, в каноне есть (W5–6: self + peer-review, светофор, ментор на 🟡/🔴). Хороший аргумент «вас не бросят на полпути».

### 16. Сертификат, чат бессрочно, тарифы 39/49k, возврат, рассрочка
Канона не касаются — решает бизнес. Противоречий с программой нет.
